13. Toolbox

13.1. Flash testing

  • FlexPilot

    FlexPilot is a Selenium extension that allows Flex-based testing. It requires a modification of the app under test by inclusion of a library and definition of the functions to expose (homepage: http://wiki.github.com/mde/flex-pilot). It is currently usable only through FlexPilotX, a Selenium IDE extension.

  • FlexUISelenium

    This Selenium RC extension gives the possibility to test Flex UI components and functions. It is only available for Java-based tests and also requires a modification of the target app (homepage: http://code.google.com/p/flex-ui-selenium).

13.2. Selenium helper tools

  • Firebug

    Firebug is without doubt the most efficient and famous debugging tool for web developers. It is actually a Firefox extension and offers:

    • HTML inspection
    • DOM inspection
    • CSS edition on-the-fly
    • JavaScript debugger
    • Traffic analyzer

    Google Chrome offers a similar tool that comes with the browser

  • Firefinder

    Firefinder is a Firebug extension provided as a Firefox extension that adds a CSS and XPath-based element lookup feature

  • XPather

    Xpather is another Firefox extension useful to generate XPath patterns just by right-clicking the target element

  • Stored Variables Viewer

    Stored Variables Viewer is a Selenium IDE extension that provides the capability to "watch" the stored variables used in the tests. It installs as a Firefox extension.

13.3. Test developer tools

  • CubicTest

    • CubicTest is an Eclipse plugin made to write Selenium and watir tests
    • You write tests by describing your pages, their state and transition between them
    • You can directly execute tests from the IDE
    • It is however a basic tool that doesn’t support loops and conditions

  • RC from command-line

    • If you’re using Selenium with a language that provides an interpreter, you may find it useful for debugging purposes to set-up an RC environment usable from this interpreter

    • An examble is provided in the labs/lab6/boot.rb file

      • To use it, first launch irb, the ruby interpreter
      • In irb, use require 'boot.rb' command to load the environment
      • You can start RC with Page.start_session
      • You can now use all the API commands to interact with you application
